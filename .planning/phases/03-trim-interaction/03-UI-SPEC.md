## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — Phase 3 has no submit action. Trim execution is Phase 4. |
| "Cut from start" label | "Cut from start (s)" |
| "Cut from end" label | "Cut from end (s)" |
| Input placeholder | "0.00" |
| Keyboard hint | "Arrow keys: ±0.1s  Shift+Arrow: ±1.0s" |
| Empty state (no file loaded) | Controls are not rendered — waveform placeholder reads "Load a file to see the waveform" (established Phase 2) |
| Invalid state (start would exceed end) | Inputs are clamped silently — no error toast. Input snaps to maximum valid value. No copy needed. |
| Decoding in progress | "Decoding audio..." (established Phase 2 — TrimControls not rendered during this state) |

**Copy rationale:**
- Labels use "(s)" suffix for seconds — compact and universally understood in audio tools.
- "Cut from start" and "Cut from end" match the REQUIREMENTS.md specification (WAVE-03) exactly.
- The keyboard hint is rendered as a `<p>` below the inputs, 12px, `--text` color. It is always visible when a file is loaded, not only on focus.
- No destructive actions exist in Phase 3. No confirmation copy needed.

**Source:** REQUIREMENTS.md WAVE-03 specifies "seconds to cut from start/end" as the input model. CONTEXT.md defers all label/format decisions to Claude's discretion.

## Interaction States

### Numeric Input States

| State | Visual |
|-------|--------|
| Default | Border `var(--border)`, no shadow |
| Focus | Border `var(--accent)`, shadow `0 0 0 3px var(--accent-bg)` |
| Hover | Border `color-mix(in srgb, var(--border) 70%, var(--accent) 30%)` — subtle approach toward accent |
| Disabled | Not applicable — inputs are always active when a file is loaded |
| Clamped (value snapped) | No visual change — clamping is silent; input displays the snapped value |

### Waveform Handle States

| State | Visual |
|-------|--------|
| Default | Regions plugin renders handles; no custom override needed |
| Dragging | Browser sets `cursor: ew-resize` on the handle element (Regions plugin default) |
| During drag | Region fill expands/contracts in real time (Regions plugin) |
| After drag release | Numeric inputs update to reflect new position (via store) |

### Keyboard Nudge

- `Arrow keys` (when input focused): ±0.1s increment (native `step="0.1"` behavior)
- `Shift + Arrow keys` (when input focused): ±1.0s increment (custom `onKeyDown` handler)
- No visible indicator change beyond the input value updating (which drives waveform region via store)

## Layout Placement

TrimControls renders directly below the waveform container, separated by 16px margin. App.tsx render order:

```
<div class="app-container">
  <h1>WebP Trimmer</h1>
  <FileLoader />
  {error && <div class="error-box">...</div>}
  {decoding && <p class="decoding-text">Decoding audio...</p>}
  {ready && <div class="file-info">...</div>}
  <WaveformView />           ← already exists
  {ready && <TrimControls />} ← NEW: only shown when file is loaded
</div>
```

TrimControls is only rendered when `status === 'ready'` (file loaded and decoded).
