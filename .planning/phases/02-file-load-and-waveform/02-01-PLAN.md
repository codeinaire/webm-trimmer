Create `src/store/trimStore.ts`:
- Import `create` from `zustand`
- Define `AppStatus` type: `'idle' | 'decoding' | 'ready' | 'error'`
- Define `TrimStore` interface with Phase 2 fields: `file: File | null`, `audioBuffer: AudioBuffer | null`, `duration: number`, `status: AppStatus`, `errorMessage: string | null`
- Add Phase 3 stubs: `trimStart: number` (default 0), `trimEnd: number` (default 0)
- Add Phase 4 stub: `outputBlob: Blob | null` (default null)
- Actions: `setFile(file: File)` sets file and status='decoding' and clears errorMessage; `setAudioBuffer(buf: AudioBuffer)` sets audioBuffer, duration=buf.duration, trimEnd=buf.duration, status='ready'; `setStatus(status, error?)` sets status and errorMessage; `reset()` resets all fields to defaults
- Export `useTrimStore`

Create `src/utils/formatValidation.ts`:
- Define constants: `WEBM_MAGIC = [0x1a, 0x45, 0xdf, 0xa3]` and `RIFF_MAGIC = [0x52, 0x49, 0x46, 0x46]`
- Export type `FormatCheckResult = { valid: true; container: 'webm' } | { valid: false; container: 'webp' | 'riff' | 'unknown'; reason: string }`
- Export async function `checkFileFormat(file: File): Promise<FormatCheckResult>`
  - Read first 4 bytes via `file.slice(0, 4).arrayBuffer()`
  - Check against WEBM_MAGIC: return `{ valid: true, container: 'webm' }`
  - Check against RIFF_MAGIC: return `{ valid: false, container: 'webp', reason: 'This file uses the RIFF/WebP container format, which does not support audio. Load a .webm file recorded by Chrome or another WebM-compatible tool.' }`
  - Otherwise: return `{ valid: false, container: 'unknown', reason: 'Unrecognised file format. Load a .webm audio file.' }` (include hex of first 4 bytes in the message)
- Also add a file size guard: export `MAX_FILE_SIZE = 50 * 1024 * 1024` (50 MB) — callers check before decode
  </action>

Create `src/services/audioDecoder.ts`:
- Private module-level variable: `let audioCtx: AudioContext | null = null`
- Private function `getAudioContext(): AudioContext` — creates AudioContext lazily (only on first call, after user gesture). If existing context state is 'closed', create a new one.
- Export async function `decodeForWaveform(arrayBuffer: ArrayBuffer): Promise<AudioBuffer>`
  - Calls `getAudioContext()`
  - Calls `ctx.decodeAudioData(arrayBuffer.slice(0))` — CRITICAL: `slice(0)` creates a copy so the original ArrayBuffer is NOT detached. The original buffer must remain usable for Phase 4 ffmpeg.wasm.
  - Returns the resulting AudioBuffer
- Do NOT create AudioContext at module load time (browser blocks it before user gesture)
- Do NOT use the original arrayBuffer directly (decodeAudioData detaches it)
  </action>
