## Color

All values from `src/index.css` CSS custom properties. Dark mode overrides are automatic.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#fff` (`--bg`) | `#16171d` | Page background, input backgrounds |
| Secondary (30%) | `#f4f3ec` (`--code-bg`) | `#1f2028` | `.trim-controls` background, `.trim-actions` background |
| Accent (10%) | `#aa3bff` (`--accent`) | `#c084fc` | Focus rings on inputs, Trim button background |
| Destructive | n/a | n/a | No destructive actions in this phase |

Accent reserved for:
- Trim button background (primary CTA only — not the Download button)
- Input focus ring border-color and box-shadow (existing `.trim-input:focus` pattern)

Download button uses secondary surface styling: `background: var(--code-bg)`, `border: 1px solid var(--border)`, `color: var(--text-h)`. It is a secondary action that appears only after trim succeeds.

Disabled state: Trim button uses `opacity: 0.45` with `cursor: not-allowed` — no color override, preserves accent color at reduced opacity.

Processing state: Trim button text changes to "Trimming…" and is disabled. No color change — opacity and cursor sufficient.

Error state: Reuse existing `.error-box` — `background: #fff0f0`, `border: 1px solid #f5a5a5`, `color: #c00`.

Source: `src/index.css` CSS variables, `src/App.css` `.error-box`, `.file-loader-button`, `.trim-input:focus`.

## Component Inventory

New components and CSS classes required for Phase 4:

### `TrimActions` component (`src/components/TrimActions.tsx`)

Renders below `<TrimControls />` in App.tsx. Contains:

1. **Estimated size line** — visible when file is loaded AND trim is not a no-op (trimStart > 0 or trimEnd < duration)
   - CSS class: `.size-estimate`
   - Copy: `Estimated: {X KB}`
   - Hidden when trim is full duration (prevents confusing "Estimated: 252 KB" === original)

2. **Trim button** — primary CTA
   - CSS class: `.trim-button`
   - States: default, hover, disabled (no-op), disabled (processing / "Trimming…")
   - Accent background, full border-radius 6px, height 40px, horizontal padding 24px

3. **Output panel** — visible only when `outputBlob !== null`
   - CSS class: `.output-panel`
   - Contains: `.size-comparison` text + `.download-button`

4. **Size comparison** — inside output panel
   - CSS class: `.size-comparison`
   - Copy: `Original: {X KB} → Trimmed: {Y KB} ({Z}% smaller)`
   - 14px / weight 400 / color: `var(--text-h)`

5. **Download button** — secondary CTA, inside output panel
   - CSS class: `.download-button`
   - Secondary surface styling (not accent)
   - Copy: `Download`

### App.css additions

New classes: `.trim-actions`, `.trim-button`, `.size-estimate`, `.output-panel`, `.size-comparison`, `.download-button`, `.file-sep`.

The `.trim-actions` block mirrors `.trim-controls` layout: same padding (16px), same margin-top (16px), same border-radius (6px), same background (`var(--code-bg)`), same border (`1px solid var(--border)`).

### App.tsx file-info extension

Add `file.size` display after duration: `{file.name} • {duration.toFixed(1)}s • {formatBytes(file.size)}`

Uses existing `.file-info` flex layout. Add `.file-sep` class for bullet separators (color: `var(--border)`, acts as visual delimiter).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (trim button — idle) | `Trim` |
| Primary CTA (trim button — processing) | `Trimming…` |
| Secondary CTA (download button) | `Download` |
| File info size display | `{filename} • {N.N}s • {X KB}` |
| Estimated size (pre-trim, while dragging) | `Estimated: {X KB}` |
| Size comparison (post-trim, output panel) | `Original: {X KB} → Trimmed: {Y KB} ({Z}% smaller)` |
| Processing state (no separate text — button label changes) | `Trimming…` (button label) |
| Error state (trim failure) | `Trim failed — please try again.` (inside `.error-box`, reuses existing component) |
| Error state (console detail) | Raw error logged to `console.error`, not shown to user |
| No file loaded (waveform placeholder, existing) | `Load a file to see the waveform` (existing — unchanged) |

Destructive actions: None. No confirmation required. The Trim button replaces the previous result (not destructive — user can re-trim any time). The "moving handles clears result" state transition is silent — output panel disappears automatically, no confirmation dialog.

Size formatting rule: `< 1 MB → show KB with 1 decimal (e.g., "252.0 KB")`. `>= 1 MB → show MB with 1 decimal (e.g., "1.2 MB")`. Function: `formatBytes(bytes: number): string` in `src/utils/formatBytes.ts`.

Source: CONTEXT.md decisions — two-step flow, exact format of size comparison string confirmed by user (`Original: 252 KB → Trimmed: 180 KB (29% smaller)`).

## Interaction States

### Trim Button State Machine

| State | Condition | Visual |
|-------|-----------|--------|
| Disabled (no-op) | `trimStart === 0 && trimEnd === duration` | Accent bg, opacity 0.45, cursor not-allowed |
| Enabled | trim region is not full duration | Accent bg, full opacity, pointer cursor |
| Processing | `isProcessing === true` | Accent bg, opacity 0.45, cursor not-allowed, label "Trimming…" |
| Re-trim (after first trim) | `outputBlob !== null` and trim enabled | Same as "Enabled" — no label change |

Source: CONTEXT.md — "Trim button disabled when start=0 and end=duration" and "Button disabled during processing".

### Output Panel Visibility

| State | Output Panel |
|-------|-------------|
| No trim performed yet | Hidden |
| Trim complete | Visible: size comparison + Download button |
| Handles moved after trim | Hidden (outputBlob cleared to null by store action) |
| Processing new trim | Hidden (outputBlob cleared at start of trim) |

Source: CONTEXT.md — "Moving handles after a trim clears the previous result".

### Estimated Size Visibility

| State | Estimated Size Line |
|-------|---------------------|
| No file loaded | Hidden |
| File loaded, full duration selected (no-op) | Hidden (avoids showing "Estimated: 252 KB" when no trim is active) |
| File loaded, trim region narrower than full duration | Visible: `Estimated: {X KB}` |
| Processing | Hidden (irrelevant during active trim) |
| Output panel visible | Hidden (replaced by actual size comparison) |

Source: RESEARCH.md Pitfall 5 — "Only show the estimated size line when the trim region is NOT the full file".

## Layout

Top-to-bottom flow in `App.tsx` (locked in CONTEXT.md):

```
<div class="app-container">
  <h1>WebP Trimmer</h1>
  <FileLoader />
  [error-box if error]
  [decoding-text if decoding]
  [file-info if ready: name • duration • size]   ← extend with file size (Phase 4)
  <WaveformView />
  [TrimControls if ready]                        ← existing, unchanged
  [TrimActions if ready]                         ← NEW — trim button, size display, download
</div>
```

`TrimActions` is a separate component from `TrimControls` — executor must not merge them. It lives below the numeric inputs at the bottom of the control flow.
