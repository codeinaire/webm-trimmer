1. Add to the `TrimStore` interface:
```
setTrimStart: (n: number) => void
setTrimEnd: (n: number) => void
```

2. Add to the `create()` body after the existing `reset` action:
```typescript
setTrimStart: (n) =>
  set((s) => ({ trimStart: Math.max(0, Math.min(n, s.trimEnd - 0.01)) })),
setTrimEnd: (n) =>
  set((s) => ({ trimEnd: Math.max(s.trimStart + 0.01, Math.min(n, s.duration)) })),
```

The clamping logic enforces:
- `trimStart` is always >= 0 and always at least 0.01s before `trimEnd`
- `trimEnd` is always at least 0.01s after `trimStart` and never exceeds `duration`
- This prevents any invalid state where start >= end

Do NOT modify existing actions (setFile, setAudioBuffer, setStatus, reset). Only add the two new actions.

**Effect 2: Store-to-region sync (dependencies: [trimStart, trimEnd])**

```typescript
useEffect(() => {
  if (!regionRef.current) return
  isSyncingFromStore.current = true
  regionRef.current.setOptions({ start: trimStart, end: trimEnd })
  isSyncingFromStore.current = false
}, [trimStart, trimEnd])
```

This effect moves the waveform handles when trimStart/trimEnd change externally (e.g., from numeric inputs in Plan 02). The `isSyncingFromStore` guard prevents the `region-updated` event handler from writing back to the store when `setOptions` is the source.
